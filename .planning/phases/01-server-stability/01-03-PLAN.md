# Plan 01-03: Standardize Logging and Metrics

## Goal
Establish a uniform logging and observability pattern across all Synapse server components to facilitate easier debugging and future monitoring integration.

## Context
Current logging is inconsistent. Some components use `ctx.info()`, others use standard `logger.info()`, and some use both. We need a standardized approach where technical logs go to the server logs and user-facing status updates go to `ctx.info()`.

## Tasks
- [ ] Define a `SynapseLogger` utility or standardized pattern for component-level logging.
- [ ] Audit all components (`KnowledgeGraph`, `WikiAdapter`, `InsightEngine`, `TextProcessor`) for logging consistency.
- [ ] Ensure all tool operations log start, completion, and duration.
- [ ] Add basic performance metrics (e.g., execution time) for expensive graph queries and semantic processing.

## Verification
- [ ] Verify that server logs show clear, structured component status.
- [ ] Verify that `ctx.info()` is used exclusively for progress updates visible to the user/LLM.
- [ ] Run `tests/test_basic.py` and `tests/test_health.py` to ensure no regressions.
